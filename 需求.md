# 我需要写一个frpc的web管理网站（与frpc运行在同一主机）


## 需要实现以下功能
### 1.能够看到所有的配置信息，包括 tcp，udp，并且名称都需要看到
### 2.标记出所有活动的通道
可以通过  http://127.0.0.1:7400/api/status 
这个get请求获取，数据如下：
{"tcp":[{"name":"145","type":"tcp","status":"running","err":"","local_addr":"192.168.18.145:3389","plugin":"","remote_addr":"soft.mybips.com:6005"},{"name":"e450-001","type":"tcp","status":"running","err":"","local_addr":"192.168.0.198:3389","plugin":"","remote_addr":"soft.mybips.com:6003"},{"name":"rdp","type":"tcp","status":"running","err":"","local_addr":"192.168.0.87:5938","plugin":"","remote_addr":"soft.mybips.com:5938"}],"udp":[{"name":"rdp-udp","type":"udp","status":"running","err":"","local_addr":"192.168.0.87:5938","plugin":"","remote_addr":"soft.mybips.com:5938"}]}

### 3.增加配置信息启用，停用功能。可以设置某一个通道启用（重新修改frpc.toml文件并且frpc reload -c frpc.toml）

### 4.管理网站需要增加用户认证功能，只有登录的用户才可以执行frpc操作。
### 5.前端使用vue3，后台使用.net8,数据库使用sqlite

### 6.可以增加通道，增加通道后，显示在列表中。点击列表启用按钮，此时写入到 frpc.toml文件并且frpc reload -c frpc.toml

增加通道需要描述好 各字段名称，作用。
